## V2B+ — Diff Explainer (LLM microservice)
## ════════════════════════════════════════════════════

- ⏳ New module: `constant-diff-explainer` — Quarkus + GraalVM native microservice      P2f
  → Embeds llama.cpp (via JNI or Panama FFI) + a quantised model (e.g. Mistral-7B-Q4_K_M)
  → Exposes POST /explain  { diff: DiffResult } → { explanation: string }
  → GPU acceleration: detect CUDA / Metal / Vulkan at startup; fall back to CPU if unavailable
  → GraalVM native-image: bundle model weights as a resource or mount at runtime via
     QUARKUS_EXPLAINER_MODEL_PATH env var (avoid baking large binary into the image)
  → Docker Compose: add `explainer` service (image built with --features=gpu flag toggle);
     app calls it over internal app_network only
  → UI: "Explain diff" button in DiffViewer.tsx — calls /api/explain, shows markdown tooltip
  → Auth: inherit Bearer token forwarded from constant-tracker-app (no separate login)
  → Fallback: if explainer service is unreachable, button is disabled with a tooltip
